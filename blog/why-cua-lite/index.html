Why CUA-Lite — and a call for contributors.

Why CUA-Lite

Computer-use agent resources are fragmented. Every benchmark ships its own runner, action space, and data format. Every dataset picks its own schema, so data collected for one agent can't train another. Every project rebuilds the same plumbing — screenshot loops, VM setup, eval scripts — before a single rollout runs.

CUA-Lite puts all of it in one place — all a computer-use agent needs, open to share, simple to run: (1) unified data, 10+ datasets and fresh GPT-5.5 rollouts, free on Hugging Face; (2) one framework to eval, SFT, and RL any agent across desktop, web, and mobile. Here is how each contribution compounds:

Datasets

Convert a dataset once, and every agent can train on it. LiteSample is the one schema, shared across every env, agent, and task type, free on Hugging Face.

Every model wants your training data packed its own way — CUA-Lite ships a ready harness for each. That's the adapter in the figure above: it packs a LiteSample row into the exact shape a model trains on, with the building blocks to add your own. The next figure plays out Qwen3.5's.

Call for data contributors. Data collected for one agent won't train another out of the box. In LiteSample it trains every agent — even models that don't exist yet. One conversion, and the whole community can train on it.

Environments & agents

Add an env, and every agent runs it; add an agent, and it runs every env. They meet in lite.gym — screenshots flow up, actions flow down, in one action space plus each env's extra tools. The same loop serves eval and RL: every env ships eval tasks, and many add training splits with verifiable rewards.

Call for env contributors. An env only matters while people run it. Behind lite.gym, yours runs against every agent, now and later, on one shared leaderboard. One integration, and the whole field trains and competes on it.
