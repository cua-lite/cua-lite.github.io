Why CUA-Lite

Computer-use agent research is fragmented. Every benchmark ships its own harness, its own action space, its own data format. Every dataset picks its own schema, so data collected for one agent can't train another. Every lab rebuilds the same plumbing — screenshot loops, VM setup, eval scripts — before running a single experiment.

CUA-Lite puts all of it in one place — all a computer-use agent needs, open to share, simple to run: (1) unified data, 10+ datasets and fresh GPT-5.5 rollouts, free on Hugging Face; (2) one framework to eval, SFT, and RL any agent across desktop, web, and mobile. Here is how each contribution compounds:

Datasets

Convert a dataset once, and every agent can train on it. LiteSample is the one schema, shared across every env, agent, and task type. No data of your own? It's all free on Hugging Face.

The adapter stage above is the agent harness doing double duty: it runs the model live in the envs, and it converts standardized LiteSample rows into the data format that harness needs. This is where the training complexity hides — Qwen3.5's harness, for example, keeps the last four screenshots, collapses the older history to a fixed line of text, and radix-packs prefix-extending steps into a single forward pass; the next figure plays it out. CUA-Lite ships every built-in model's harness ready to use, plus the building blocks to compose a new one with ease.

Call for data contributors. A dataset usually dies with the model generation it was built for. In LiteSample it doesn't: models that don't exist yet will train on it, and it joins a growing commons of computer-use data — grounding steps, full trajectories, every platform. If you've collected trajectories, one conversion puts them to work for everyone.

Environments & agents

Add an env, and every agent runs it; add an agent, and it runs every env. They meet in lite.gym — screenshots flow up, actions flow down, in one action space plus each env's extra tools. The same loop serves eval and RL: every env ships eval tasks, and many add training splits with verifiable rewards.

Call for env contributors. A benchmark only matters while people run it. Behind lite.gym, yours runs against every agent we ship and every agent added later, on one leaderboard; give it a training split, and it becomes a place agents learn, not just get graded. One integration keeps it alive.
