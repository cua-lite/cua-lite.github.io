Why CUA-Lite

Computer-use agent research is fragmented. Every benchmark ships its own harness, its own action space, its own data format. Every dataset picks its own schema, so data collected for one agent can't train another. Every lab rebuilds the same plumbing — screenshot loops, VM setup, eval scripts — before running a single experiment.

CUA-Lite puts all of it in one place — all a computer-use agent needs, open to share, simple to run: (1) unified data, 10+ datasets and fresh GPT-5.5 rollouts, free on Hugging Face; (2) one framework to eval, SFT, and RL any agent across desktop, web, and mobile. Here is how each contribution compounds:

Datasets

Convert a dataset once, and every agent can train on it. LiteSample is the one schema shared across every env, agent, and task type; each agent's adapter then turns the same rows into that model's own scaffolding. No data of your own? It's all free on Hugging Face.

Environments & agents

Integrate an env once, and every agent can run it; integrate an agent once, and it runs every env. They meet in lite.gym — screenshots flow up, actions flow down, in one action space plus each env's extra tools. The same loop serves eval and RL: OSWorld ships 369 eval tasks; Lite.OSWorld adds 2,490 generated training tasks with verifiable rewards. Scores land on the leaderboard as committed JSON — an eval you already ran is worth submitting.
