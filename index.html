Quickstart

A trajectory in five lines.

Sample a rollout with any computer-use agent. What comes back is a LiteSample — ready to evaluate, SFT, or RL.

quick_start.py
$ uv sync --extra quick-start>>> import lite.gym as gym, lite.agents as agents>>> env = gym.make("@")>>> agent = agents.make("", env=env)>>> result = await agent.sample(env) # → LiteSample

01 · Unified data

One schema, any data.

One format — LiteSample — for any data, shared across every env, agent, and task type; per-agent adapters then postprocess it into each model's own scaffolding, ready to SFT or RL. It fills from two sources:

SFT data at scale 🤗 Corpora 10+ existing CUA datasets — grounding · understanding · use — preprocessed, ready to fine-tune any agent. Browse collection ↗ SFT data on demand Rollouts Roll out any teacher (GPT-5.5, Claude…) into the same format and distill into any student. We ship the pipeline, not just the data. run_sft.sh
# --- host ---
# 1 · download the corpus
$ python -m lite.data.hf.download \
--out .data/hf/cua-lite/

# 2 · export a model-ready SFT parquet
$ python -m lite.train.export.export_sft \
--model-id \
--config scripts/configs//recipes/sft/default.yaml \
--data-paths .data/hf/cua-lite/ \
--image-root .data/hf \
-o .data/sft//.parquet

# --- Slime container (see docs/slime.md) ---
# 3 · supervised fine-tune
$ MODEL_ID= \
PROMPT_DATA=.data/sft//.parquet \
bash scripts/train/run_sft.sh
